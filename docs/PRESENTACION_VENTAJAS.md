# ALSUA vs XML en línea / otras herramientas  
Lista para presentación

> Enfoque del producto: Web Service SAT + export Almacén Digital CONTPAQi + auditoría.

---

## 1. Ventajas frente a CONTPAQi XML en línea+

| # | Ventaja ALSUA | Por qué importa |
|---|---------------|-----------------|
| 1 | **Hecho para Contabilidad / ADD** | Exporta carpetas/ZIP listos para *Analizar directorio* → Importar. Menos fricción que gestionar todo en otro producto CONTPAQi. |
| 2 | **Tus XML en tu disco** | Los archivos quedan en tu servidor/PC. No dependes de la nube del proveedor para recuperar histórico. |
| 3 | **Sin candado de ecosistema** | Puedes usar CONTPAQi Contabilidad y, a la vez, Excel, auditoría propia o otros sistemas. |
| 4 | **Automatizable (CLI + scheduler)** | Sync diario/scripts para despacho; no solo “clic en pantalla”. |
| 5 | **UI propia con marca ALSUA** | Experiencia del despacho/empresa, no la interfaz genérica de un tercero. |
| 6 | **Auditoría incluida en el flujo** | Faltantes, cancelados y 69-B junto a la descarga — no solo bajar y ver. |
| 7 | **Control de costos a largo plazo** | Evita dependencia de licencias/renovaciones cloud si el volumen es alto y el flujo ya es interno. |

### Frases cortas (bullet slides)

## 2. Ventajas frente al portal SAT (visor / descarga web)

| # | Ventaja | Detalle |
|---|---------|---------|
| 1 | Alto volumen | Hasta ~**200,000** CFDI por solicitud vía WS vs ~**2,000/día** en portal |
| 2 | Sin captchas ni clics | Proceso automatizable con e.firma |
| 3 | Metadata + XML | Cruces y control de cancelados |
| 4 | Historial operable | Cola, reanudación, dedupe UUID |
| 5 | Entrega contable | Paquete listo para CONTPAQi, no solo ZIP suelto |

---

## 3. Ventajas frente a otros descargadores (DesMas, Facturando, iAudita, etc.)

| # | Ventaja ALSUA | Diferenciador |
|---|---------------|---------------|
| 1 | **Especialización CONTPAQi Contabilidad** | El “último kilómetro” es el ADD, no solo acumular XML |
| 2 | **Operación local / on-premise** | Ideal si el cliente no quiere sacar FIEL/XML a SaaS externo |
| 3 | **Stack abierto y documentado** | Python + API; editable a la medida del despacho |
| 4 | **CLI + UI** | Mismo motor para humano y para automatización |
| 5 | **Enfoque WS puro** | Menos dependencia de scrapers del portal (más frágiles) |
| 6 | **Roadmap de cumplimiento** | Metadata, 69-B, cancelados y faltantes en el mismo producto |
